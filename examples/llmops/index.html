Before · 上线初期

prompt 硬编码
在代码里

  • 回复质量忽高忽低

  • 没人看见出了什么问题

After · 引入 LLMOps

  • 低分自动触发人工审核
  • 客服满意度明显回升

Action · 今天检查一件事

你的 prompt
写在哪?

A

写死在代码里改一次 = 发一次版,这是缺 LLMOps 的最明显信号。

B

配置文件里能改,但没人评估输出质量,算半步。

C

专门的平台prompt、评分、成本、安全都在一处管理。

答案是 A?从把 prompt 拿出代码开始 —— 这就是引入 LLMOps 的第一步。
